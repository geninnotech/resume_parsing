you are given a `resume ocr text` along with a `resume structured json` your task is to Populate the structured json with the available ocr text. 

some important thigns
1. **DATE HANDLING ‚Äî ‚ÄúPresent / Current / Till date‚Äù**

    * For **any** date range (experience, education, volunteering, certifications, licenses, publications, patents, awards) that indicates ‚ÄúPresent/Current/Till date/ongoing‚Äù, set:

    * `end_date.month` = `""` (empty string)
    * `end_date.year`  = `"Present"`
    * **Never** put ‚ÄúPresent‚Äù in the `month` field. It **must** go in the `year` field.
    * If only a year is given, set `month` to `""` and `year` to that year as a string (e.g., `"2021"`).
    * Record months in numbers only if any other format is provided (e.g., `"Jun"`, `"June"` `"Jan"`, `"January"`). then convert them to respective numbers. So `"Aug"` or `"August"` becomes `8`.

2. NORMALIZATION RULES 

* **Currency** : if any salary or compensation expectation is mentioned or current salary is mentioned, and if any currency type is not specified, then assume the currency to be INR. if if the currency type is specifically mentioned then you should use that currency type and do not convert the values.

* **Currency type** : always mention the currency type in `ISO 4217 standard currency codes` format, such as USD, INR.

* **Numbers with K/M**: If a **single** compensation/salary figure appears as ‚Äú150k‚Äù or ‚Äú1.2M‚Äù, convert to integer if unambiguous:

  * `k` = √ó1000; `m`/`M` = √ó1000000; strip currency symbols and commas.
  * `Lakh` or `L` means Lakh that is 100000.
  * If the value is a **range** or unclear, set to `null`.
* **Phone numbers**: capture exactly as printed (including country codes, spaces, dashes); if extension is shown (e.g., ‚Äúext 234‚Äù, ‚Äúx234‚Äù), put the numeric part in `"ext"`; otherwise `"ext": ""`.

3. **Categorical fields** (e.g., `site_preference`, `employement_type`, `work_time_preference`, `languages[].proficiency`, `experience[].employment_type`):

  * Use the exact allowed values shown in the schema comments.
  * you can infere these values, to best of your knowledge, you can give benefit of doubt only in this case.
  * But, you should not make up the field. you can **only** infer the values based on the real values given.
  * eg. if language proficiency is given in the resume as intermediate, you can infer that the proficiency for that Categorical value should be 5 out of 10. But if no proficiency value is given in the resume, then you should just give the null which is the default value.
